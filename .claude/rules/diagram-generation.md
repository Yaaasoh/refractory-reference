# 図生成の行動指針

**適用範囲**: SVG、Mermaid、HTML+CSS、Graphvizによる図の生成全般
**出典**: facility-safety Phase 2調査（28件）の知見抽出

## 原則1: テンプレート編集方式

ゼロから図を生成しない。既存テンプレートまたは参考コードを編集する。

- テンプレートがあればコピーしてテキスト内容のみ書き換え
- テンプレートがなければ、類似する既存の図をベースにする
- レイアウト・色・フォントは既存定義を維持する

## 原則2: 段階的詳細化（4フェーズ）

1回で完成を目指さない。構造→レイアウト→スタイル→微調整の順に進む。

1. **構造**: 要素と接続関係のみ（位置・色は後回し）
2. **レイアウト**: 配置・間隔・整列の調整
3. **スタイル**: カラーパレット・フォント・線種の適用
4. **微調整**: テキスト折り返し・矢印・タイトルの修正

## 原則3: 制約を先に定義

図の生成コードを書く前に、以下を必ず決定する:

- **viewBox / 描画領域**: 例 `0 0 800 600`
- **最大要素数**: 9以下（7+-2ルール）。超える場合は図を分割
- **色数**: 5色以内。CSS変数またはパレット定数で管理
- **フォント**: 1種のみ（CJKの場合は `Noto Sans JP, sans-serif`）
- **グリッド**: 座標は100刻み。中途半端な小数座標を使わない

## 原則4: LLM失敗パターンの回避

| 失敗 | 回避策 |
|------|--------|
| 座標計算ミス | グリッドベース（100刻み）で配置 |
| 要素の重なり | 1図9要素以下。超えたら分割 |
| テキストはみ出し | foreignObject+CSS折り返し、またはラベル文字数制限 |
| 色ドリフト | CSS変数で色を定数管理。修正時も「パレット維持」を明示 |
| 過剰な要素生成 | 「厳密にN個の要素のみ」と明示 |
| 構文エラーループ | 3回修正して直らなければ最初から書き直し |

## 原則5: デザインレビュー（CRAP原則）

生成後、以下4点で自己レビューする:

- **Contrast**: 重要な要素は大きく・濃く。些細な違いは避け、大胆に差をつける
- **Repetition**: 同レベルの要素は同じスタイル。色・形・線種を統一
- **Alignment**: 全要素をグリッドに揃える。ノードの上端・中心を整列
- **Proximity**: 関連要素は近く、無関連要素は離す。余白でグループを表現

## 原則6: アクセシビリティ

- 色だけに頼らない。形状・線種・テキストラベルでも情報を区別する
- コントラスト比: テキスト4.5:1以上、非テキスト3:1以上（WCAG AA）
- 装飾は最小限（影なし、グラデーションなし、3D効果なし）

## 図種×手法の選択

| 図種 | 第1推奨 | 条件 |
|------|---------|------|
| フローチャート（短ラベル） | Mermaid | 自動レイアウトが有利 |
| CJK長文ラベルの図 | HTML+CSS | テキスト折り返しが確実 |
| 幾何学的図形 | インラインSVG | 座標制御が必要 |
| グラフ構造（英語） | Graphviz | 自動レイアウト |
| 上記以外 | HTML+CSS | 最も安定 |

**詳細な手順・プロンプト例はskill `diagram-generator` を参照。**

## 検証プロセス（2段構成）

| 段階 | タイミング | 内容 | 実装 |
|:----:|-----------|------|------|
| Tier 1 | 図ファイル保存時 | L1構文+L2品質の自動警告 | `diagram_quality_check.sh` (PostToolUse) |
| Tier 2 | 完了報告時 | 視覚検証証拠の要求 | `prevent_false_completion.sh` (UserPromptSubmit) |
| Tier 3 | 任意 | CRAP原則+技術品質の網羅的レビュー | `diagram-verifier`サブエージェント |

- Tier 1-2は自動実行（hook）。Tier 3はオンデマンド
- **視覚検証は人間が実施する**。コードレベルの検査だけでは図の品質は保証できない

## 防御層（Multi-layer Defense）

| Layer | ツール | 効果 |
|-------|--------|------|
| Rules | diagram-generation.md（本文書） | 弱（原則提示） |

| Hooks | prevent_false_completion.sh (UserPromptSubmit) | 強（視覚検証ゲート） |
| Agents | diagram-verifier | 中（オンデマンドレビュー） |
