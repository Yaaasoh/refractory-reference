# テストプロセス要件

**作成日**: 2026-01-06
**目的**: 実装後の動作確認必須化
**重要度**: CRITICAL

---

## 完了の定義

### 正しい完了基準（必須）

```
✅ コードを書いた
✅ テストを実行した
✅ テストが成功した
✅ 実際の動作を確認した
→ 完了
```

### 誤った完了基準（禁止）

```
❌ コードを書いた → 完了
❌ ファイルを保存した → 完了
❌ git commitした → 完了
```

---

## 実装タイプ別テスト要件

### Webフロントエンド（HTML/CSS/JS）

| 確認項目 | 方法 |
|---------|------|
| ページ読み込み | ブラウザで開いてエラーなし |
| JavaScript実行 | コンソールにエラーなし |
| 機能動作 | 主要機能をクリック/入力して確認 |
| レスポンシブ | 画面サイズ変更して確認 |

### バックエンド（Python/Node.js等）

| 確認項目 | 方法 |
|---------|------|
| 構文エラー | `python -m py_compile file.py` |
| ユニットテスト | `pytest` / `npm test` |
| 統合テスト | エンドポイント呼び出し確認 |
| エラーハンドリング | 異常系入力でのテスト |

### シェルスクリプト

| 確認項目 | 方法 |
|---------|------|
| 構文チェック | `bash -n script.sh` |
| 実行テスト | 実際に実行して確認 |
| エラーケース | 不正入力での動作確認 |

---

## テスト失敗時の対応

### 禁止事項

1. **テスト失敗を隠して完了報告**
2. **「多分動く」での完了宣言**
3. **テストなしでの提供**

### 必須事項

1. テスト失敗の原因を特定
2. 修正を実施
3. 再テストで成功を確認
4. 成功後に完了報告
