## 2. 償いの具体的対策（System-Based Prevention）

### 2.1 データ保存チェックリスト自動生成

**対策**: WebSearch/WebFetch実行時、自動的にチェックリストを生成

**実装**:
- `.claude/hooks/`に`post-websearch.sh`配置
- WebSearch/WebFetch実行後、`DATA_SAVING_CHECKLIST.md`を自動生成
- 未保存データがあれば警告表示

**チェック項目**:
```markdown
## WebSearch/WebFetch Data Saving Checklist

### Search Executed: [YYYY-MM-DD HH:MM]
- [ ] WebSearch結果を.mdファイルに保存したか？
- [ ] WebFetchで取得したデータを保存したか？
- [ ] ダウンロードリンクを記録したか？
- [ ] 実ファイル（.par等）をダウンロードしたか？
- [ ] source.txtにURL・アクセス日を記録したか？
- [ ] README.mdに完全パスを記載したか？

⚠️ 1つでも未チェックなら**ルール違反・罪の再犯**
```

### 2.2 作業終了前の強制確認

**対策**: `.claude/settings.local.json`に作業終了前チェックを追加

**実装**:
```json
{
  "workflow": {
    "before_context_compression": [
      "check_data_saved",
      "verify_par_files_count",
      "confirm_no_claims_without_evidence"
    ]
  }
}
```

**確認内容**:
1. WebSearch/WebFetch実行回数 vs 保存ファイル数
2. 主張した機体数 vs 保存した.parファイル数
3. ドキュメントで言及したデータ vs 実ファイル存在確認

### 2.3 README.md更新の強制化

**対策**: データファイル追加時、README.md更新を強制

**実装ルール**:

- 更新内容: 完全パス、機体名、ソース、Evidence Level
- 更新しない場合 → git commitブロック

### 2.4 定期的な罪の確認（Weekly Review）

**対策**: 毎週の作業開始時、このドキュメントを再読
