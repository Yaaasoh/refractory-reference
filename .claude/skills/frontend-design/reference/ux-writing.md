# UX Writing Reference

UIテキスト・マイクロコピーの設計ガイド。

---

## ボタンラベル

**動詞 + 対象** で具体的に:

| ❌ 汎用 | ✅ 具体的 |
|---------|----------|
| 送信 | レポートをダウンロード |
| OK | 変更を保存 |
| はい | アカウントを削除 |
| 次へ | お届け先の入力へ |
| キャンセル | 編集を破棄 |

- 破壊的アクション（削除等）は赤色 + 確認ダイアログ
- プライマリボタンのラベルはダイアログタイトルに対応させる

---

## エラーメッセージ

3要素を含める: **何が起きた？ + なぜ？ + どう直す？**

| ❌ 悪い例 | ✅ 良い例 |
|----------|----------|
| エラーが発生しました | ファイルが10MBを超えています。10MB以下のファイルを選択してください |
| 入力が不正です | メールアドレスの形式が正しくありません（例: user@example.com） |
| 処理に失敗しました | サーバーに接続できません。しばらく待ってからもう一度お試しください |

**絶対禁止**: ユーザーを責める文言（「あなたが間違えました」）

## 空状態

オンボーディングの機会として活用:

```
┌──────────────────────────────┐
│                              │
│     📊 まだレポートが        │
│     ありません               │
│                              │
│  データを接続すると、        │
│  ここにレポートが表示        │
│  されます。                  │
│                              │
│  [データソースを接続]        │
│                              │
└──────────────────────────────┘
```

必須要素:
1. 状態の説明（何がないか）
2. 価値の提示（なぜ必要か）
3. アクション（次にすべきこと）

## Voice vs Tone

- **Voice（ブランドボイス）**: 常に一貫（例: 「専門的だが親しみやすい」）
- **Tone（トーン）**: コンテキストで変化

| コンテキスト | トーン |
|-------------|--------|
| オンボーディング | 歓迎、励まし |
| エラー | 共感、解決志向 |
| 成功 | 祝福、簡潔 |
| 確認ダイアログ | 明確、中立 |

---

## アクセシブルな文言

- リンク文は独立で意味が通じるように:

| ❌ 悪い例 | ✅ 良い例 |
|----------|----------|
| 詳しくは[こちら] | [料金プランの詳細]を見る |
| [クリック]して続行 | [アカウント設定]に移動 |

- `aria-label` でリンク目的を補足（必要な場合のみ）

---

## i18n考慮

- テキスト膨張を想定: 英語→日本語で20-40%増加
- 固定幅レイアウトを避ける
- 略語は避ける（翻訳困難）
- 日付・数値・通貨のフォーマットはロケール依存
- 文字列連結でメッセージを構成しない（語順が言語で異なる）

---

## 一貫性

**1概念1用語**:

| ❌ 混在 | ✅ 統一 |
|---------|---------|
| 削除/除去/消去 | 削除 |
| 設定/環境設定/オプション | 設定 |
| キャンセル/取消/中止 | キャンセル |

用語集をプロジェクトで管理し、UI全体で統一する。
