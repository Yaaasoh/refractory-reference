# コードレビュー: $ARGUMENTS

## レビュー対象

指定されたファイルまたはディレクトリをレビューします。

## レビュー観点

### 1. 正確性
- ロジックは正しいか
- エッジケースは考慮されているか
- エラーハンドリングは適切か

### 2. 可読性
- 命名は適切か
- コメントは必要十分か
- 構造は理解しやすいか

### 3. 保守性
- 重複はないか
- 依存関係は適切か
- テストは書きやすいか

### 4. セキュリティ
- 入力検証は適切か
- 機密情報の扱いは適切か
- OWASP Top 10の観点

## 出力形式

```markdown
## レビュー結果

### 重要度: 高
- [ファイル:行番号] 問題の説明
  - 推奨: 修正案

### 重要度: 中
- ...

### 重要度: 低
- ...

### 良い点
- ...
```
