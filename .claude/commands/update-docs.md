# ドキュメント更新

## 実行手順

### 1. 今回の作業で得られた知見を確認

作業内容を振り返り、以下を特定:
- 新しく学んだこと
- 発見した問題・解決策
- プロセスの改善点
- 追加すべき禁止事項・注意事項

### 2. CLAUDE.md の構造を確認

```bash
grep -n "^## " CLAUDE.md | head -20
```

**確認ポイント**:
- `## 学習メモ` セクションの有無
- 既存のセクション構成

### 3. 更新対象の決定

| CLAUDE.md の状態 | 対応 |
|-----------------|------|
| `## 学習メモ` あり | そのセクションに追記 |
| `## 学習メモ` なし | 末尾に新規セクション追加を提案 |
| CLAUDE.md なし | ユーザーに作成を提案 |

### 4. 学習メモの追記形式

```markdown
- YYYY-MM-DD: [学んだこと・教訓の要約]
  - [詳細1]
  - [詳細2]
  - **教訓**: [今後に活かすべきこと]
  - 詳細: `[関連ファイルパス]`
```

### 5. その他ドキュメントの更新確認

必要に応じて以下を更新:

| ドキュメント | 更新条件 |
|-------------|----------|
| `README.md` | プロジェクト構造の大きな変更時 |
| `MASTER_PLAN.md` | 計画の進捗・変更時 |
| 関連する `docs/*.md` | 技術的な発見・変更時 |

### 6. 更新内容の確認

```bash
git diff --stat
git diff CLAUDE.md
```

## 注意事項

- 既存の構造を壊さない
- 時系列順に追記（学習メモ）
- 具体的な日付・ファイルパスを含める
- 曖昧な表現を避ける
- セクションがない場合は追加前にユーザー確認

## 更新不要の場合

軽微な作業で新しい知見がない場合は、更新をスキップして報告:

```
ドキュメント更新: 今回の作業で特筆すべき新知見はありませんでした。
```

## 関連コマンド

- `/suggest-claude-md` - CLAUDE.md の更新候補を提案（より詳細な分析）
