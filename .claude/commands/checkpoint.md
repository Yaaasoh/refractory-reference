# /checkpoint - 進捗チェックポイント確認

30分ルールと80%閾値に基づく進捗確認を実行します。

---

## 現在の状態を確認

**タスク情報**を以下の形式で報告してください:

```
タスク名: [現在のタスク]
開始時刻: [HH:MM]
経過時間: [X分]
```

---

## 進捗評価基準（80%閾値）

| 経過時間 | 期待進捗 | 判定基準 |
|:--------:|:--------:|----------|
| 10分 | ≥30% | 設計・方針が確定しているか |
| 20分 | ≥60% | 主要部分が実装されているか |
| 30分 | ≥80% | **継続/破棄の判断ポイント** |

---

## 30分時点の判定（Popperian Approach）

```
経過30分時点での進捗:
├── ≥80%: 継続（仮説は有効）
├── 60-79%: 報告して継続を検討
└── <60%: 破棄してやり直し（仮説は反証された）
```

### 即座に破棄すべき状況

以下に該当する場合、30分を待たずに破棄を検討:

- [ ] 同じエラーを3回以上繰り返している
- [ ] 新しいアプローチが見つからない
- [ ] スコープが想定を大幅に超過している
- [ ] 外部依存でブロックされている

---

## 進捗報告テンプレート

以下の形式で現在の進捗を報告してください:

```markdown
## チェックポイント報告

**経過時間**: [X分]
**現在の進捗率**: [X%]

### 完了済み
- [完了した作業1]
- [完了した作業2]

### 残り作業
- [残り作業1]
- [残り作業2]

### 判定
- [ ] 継続（≥80%達成見込み）
- [ ] 報告して継続検討（60-79%）
- [ ] 破棄してやり直し（<60%）

### 次のアクション
[具体的な次のステップ]
```
