# 作業報告書の作成

## 対象テーマ

$ARGUMENTS

## 実行手順

### 1. リポジトリの報告書慣例を確認

既存の報告書を検索し、保存場所のパターンを把握:

```bash
find . -name "*REPORT*.md" -o -name "*INCIDENT*.md" | head -10
```

**よくある保存場所**:
- `docs/project-management/`
- `docs/reports/`
- `work/reports/`
- プロジェクト配下
- ルート直下

### 2. 保存先を決定

リポジトリの慣例に従い、適切な場所を選択。
慣例がない場合は `docs/reports/` を提案（ユーザー確認後に作成）。

### 3. 今回の作業内容を確認

```bash
git status
git diff --stat
git log --oneline -10
```

### 4. 報告書を作成

ファイル名: `WORK_REPORT_YYYYMMDD_[テーマ].md`

**報告書テンプレート**:

```markdown
# 作業報告書: [タイトル]

**日付**: YYYY-MM-DD
**作業者**: Claude [モデル名]

---

## 概要

[作業の目的と結果を1-2文で]

---

## 実施内容

### 1. [作業項目1]

- 実施内容の詳細
- 変更したファイル

### 2. [作業項目2]

- 実施内容の詳細

---

## 成果物

| ファイル | 内容 |
|----------|------|
| path/to/file | 説明 |

---

## 確認事項

- [ ] テスト/検証完了
- [ ] ドキュメント更新
- [ ] コミット準備完了

---

## 課題・残作業

（あれば記載）

---

## 次のステップ

1. [次に行うべきこと]

---

**作成日時**: YYYY-MM-DD HH:MM
```

### 5. 保存先の確認

作成前にユーザーに保存先を確認:
「報告書を `[パス]` に作成します。よろしいですか？」

## 注意事項

- インシデント報告の場合は `INCIDENT_REPORT_` プレフィックスを使用
- 引き継ぎ報告の場合は `HANDOVER_` プレフィックスを使用
- 保存先が不明な場合は必ずユーザーに確認
