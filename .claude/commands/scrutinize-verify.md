# 精査結果の事後検証

## 目的

直前の分析結果を検証し、精査として必要な要素が含まれているか確認する。
不足があれば補完する。

---

## 検証チェックリスト

### 必須項目（すべて満たすこと）

以下の3点が直前の分析に含まれているか確認:

#### 1. 問題・課題リスト

```
□ 具体的な問題が列挙されているか？
□ 各問題に根拠があるか？
□ 「特に問題なし」で終わっていないか？
  → 問題なしの場合も、確認した項目を明示
```

**不足の場合、追記**:
```markdown
## 問題・課題リスト（追記）

| # | 問題 | 重大度 | 根拠 |
|:-:|------|:------:|------|
| 1 | [問題] | 高/中/低 | [根拠] |
```

#### 2. 評価結果

```
□ 何らかの判定・評価を下しているか？
□ 評価の根拠が明確か？
□ 曖昧な表現（「良さそう」「問題なさそう」）で終わっていないか？
```

**不足の場合、追記**:
```markdown
## 評価結果（追記）

**総合評価**: [良好/要改善/問題あり]

**評価根拠**:
- [根拠1]
- [根拠2]
```

#### 3. 推奨アクション

```
□ 次に何をすべきか提示しているか？
□ アクションは具体的か？
□ 優先順位が付いているか？
```

**不足の場合、追記**:
```markdown
## 推奨アクション（追記）

| 優先度 | アクション | 理由 |
|:------:|-----------|------|
| 1 | [アクション] | [理由] |
```

---

### 推奨項目（該当すれば追記）

```
□ 重大度の判定（高/中/低）
□ カテゴリ分類
□ 影響範囲の明示
□ 時間軸（即時対応/計画的対応/将来検討）
```

---

## 検証結果の出力

検証完了後、以下のフォーマットで検証結果を報告:

```markdown
## 精査検証結果

### チェック結果

| 項目 | 状態 | 備考 |
|------|:----:|------|
| 問題・課題リスト | ✓/✗ | [補完した場合はその旨] |
| 評価結果 | ✓/✗ | [補完した場合はその旨] |
| 推奨アクション | ✓/✗ | [補完した場合はその旨] |

### 補完内容

[不足していた項目の追記内容]
```

---

## 使用場面

1. **自由な分析の後**: 形式に縛られず分析した後、品質を確保
2. **レビュー依頼前**: 分析結果を他者に渡す前の最終チェック
3. **「精査して」と言われたが形式を忘れた場合**: 事後的に補完
