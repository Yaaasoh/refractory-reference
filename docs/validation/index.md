# 品質検証システム（Validation System）

**本セクション**: 用語集の品質検証プロセスとツール

---

## 概要

本用語集は、公開前に自動検証システムによる品質チェックを実施しています。

**検証目的**:
1. **データ品質**: 文字化け・構造崩れがないこと
2. **用語集としての適切性**: 仕様値・カタログ情報ではなく、用語定義であること
3. **コンテンツの適切性**: 特定のクライアント・プロジェクトを識別できる情報がないこと

---

## 検証システム構成

### 1. 自動検証スクリプト

**ファイル**: `validate_glossary.sh`

**機能**:
- 除外パターン検索（企業名、人名、識別子）
- データ品質チェック（文字化け、構造崩れ）
- 不適切コンテンツ検出（仕様値、製品名、操業データ）

**使用方法**:
```bash
bash validation/validate_glossary.sh glossary/
```

### 2. 除外パターンファイル

**ディレクトリ**: `patterns/`

**含まれるファイル**:
- `companies.txt`: 企業名パターン
- `surnames.txt`: 人名パターン
- `identifiers.txt`: 識別子パターン

### 3. 検証ワークフロー文書

**ファイル**: `VALIDATION_WORKFLOW.md`

**内容**:
- 検証カテゴリの詳細説明
- 検証プロセスの手順
- 判定基準

### 4. 参照規格

**ファイル**: `REFERENCES.md`

**内容**:
- 用語集で参照している規格一覧
- データ品質基準

---

## 検証プロセス

### Phase 1: 自動検証

自動検証スクリプトによる機械的チェック

**実施内容**:
- 除外パターンの検索
- データ品質の確認
- 不適切コンテンツの検出

### Phase 2: 目視確認

人間による最終確認

**実施内容**:
- 自動検証の検出結果の確認
- 用語定義として適切かの判断
- 全体構成の確認

---

## 品質基準

本用語集は以下の基準で品質を維持しています:

✅ **データ品質**: 文字化け・構造崩れなし
✅ **用語集適合性**: 用語定義形式で記載
✅ **コンテンツ適切性**: 特定プロジェクト情報なし

---

## 検証結果の確認

各用語集ファイルは、上記の検証プロセスを経て公開されています。

検証の詳細については、[ワークフロー文書](VALIDATION_WORKFLOW.md)を参照してください。
